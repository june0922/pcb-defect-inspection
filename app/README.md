# PCB Defect Inspection Review Station - 사용자 매뉴얼

본 매뉴얼은 AI 모델(YOLOv8 5-Fold 앙상블)이 탐지한 PCB 결함을 작업자가 최종적으로 검토하고 판정하기 위한 **Review Station(리뷰 스테이션) 프로그램**의 사용 방법을 안내합니다.

---

## 1. 프로그램 개요
이 프로그램은 AI가 1차적으로 찾아낸 결함 후보들을 작업자가 빠르고 피로도 없이 검토할 수 있도록 설계되었습니다. 
AI의 판단 신뢰도(Confidence Score)에 따라 결함이 분류되며, 키보드 단축키를 이용해 초고속으로 불량(진짜 결함)과 정상(가짜 결함)을 판정할 수 있습니다.

---

## 2. 시작하기

1. **프로그램 실행:** `run.py`를 실행하여 프로그램을 켭니다.
2. **폴더 열기:** 
   - 좌측 상단 메뉴 바에서 **[File] > [Open Folder...]** 를 클릭합니다.
   - 검사할 이미지가 들어있는 폴더를 선택합니다. (기본값: `preprocessed_data/images/test`)
3. **AI 연산 대기:** 
   - 폴더를 선택하면 화면 하단 상태 표시줄에 진행률 바가 나타나며 AI 모델 로딩 및 앙상블 추론 연산이 시작됩니다.
   - 연산이 완료되면 화면에 이미지와 결함 내역이 표시됩니다.

---

## 3. 화면 구성 이해하기

프로그램 화면은 크게 3가지 영역으로 나뉩니다.

* **🌍 Global View (좌측 메인 화면):**
  - 현재 선택된 전체 이미지를 보여줍니다.
  - AI가 찾아낸 모든 결함 위치가 네모 박스(Bounding Box)로 표시됩니다.
  - 선택된 결함은 **하늘색 십자선(Crosshair)**으로 강조되어 어디에 위치해 있는지 한눈에 알 수 있습니다.
* **🔍 Local View (우측 상단 화면):**
  - 현재 선택된 결함 부위만 고배율로 확대해서 보여줍니다.
  - 미세한 스크래치나 찍힘 등을 정밀하게 확인할 때 사용합니다.
* **🎞️ Thumbnail Strip (하단 썸네일 리스트):**
  - AI가 현재 이미지에서 찾아낸 모든 결함들을 작은 썸네일 형태로 나열합니다.
  - 마우스로 클릭하거나 키보드 방향키를 이용해 검토할 결함을 선택할 수 있습니다.

---

## 4. 시각적 색상 가이드

작업자의 시각적 피로도를 줄이고 직관적인 판정을 돕기 위해 **결함 박스 색상**과 **썸네일 테두리 색상**이 다르게 적용됩니다.

### 4.1 이미지 결함 박스 색상 (AI 신뢰도)
AI가 얼마나 확신하는지에 따라 결함 박스의 색상이 다르게 표시됩니다.
* 🔴 **빨간색 (90% 이상):** 확실한 불량 (우선 처리 대상)
* 🟡 **노란색 (70% ~ 89%):** 주의 요망 / 모호함 (작업자의 면밀한 확인 필요)
* 🔘 **회색 (70% 미만):** 가짜 결함 의심 (빠르게 훑어보고 넘기는 대상)

### 4.2 썸네일 테두리 색상 (작업자 판정 상태)
작업자가 해당 결함을 어떻게 판정했는지 상태를 나타냅니다.
* 🔘 **회색 (Pending):** 아직 판정하지 않은 대기 상태 (기본값)
* 🔴 **빨간색 (Fail):** 작업자가 "실제 결함"으로 확정함
* 🟢 **초록색 (Pass):** 작업자가 "정상(가짜 결함, 먼지 등)"으로 확정함

---

## 5. 검토 및 판정 방법 (튜토리얼)

실제 작업 시에는 마우스보다 **키보드 단축키**를 사용하는 것이 훨씬 빠릅니다.

1. 이미지가 로드되면 하단 썸네일 리스트의 **첫 번째 결함이 자동으로 선택**됩니다.
2. 우측 상단의 **Local View(확대 화면)**를 보고 결함인지 먼지인지 판단합니다.
3. **판정하기 (단축키):**
   - ❌ 실제 결함이 맞다면? ➡ 키보드 **`Y`** (Yes)를 누릅니다. (썸네일 테두리가 **빨간색**으로 변함)
   - ✅ 먼지나 정상 패턴이라면? ➡ 키보드 **`N`** (No)을 누릅니다. (썸네일 테두리가 **초록색**으로 변함)
4. **이동하기:**
   - 키보드 **`오른쪽 방향키 (→)`** 를 누르면 다음 결함으로 넘어갑니다.
   - 키보드 **`왼쪽 방향키 (←)`** 를 누르면 이전 결함으로 돌아갑니다.
5. 이 과정을 반복하여 하단의 모든 썸네일을 회색(Pending)에서 빨간색/초록색으로 변경합니다.

---

## 6. 옵션 및 환경 설정 (Settings)

AI가 너무 많은 가짜 결함을 잡아내거나, 반대로 놓치는 결함이 있다면 필터링 옵션을 조절할 수 있습니다.
상단 메뉴에서 **[Option] > [Settings...]** 를 클릭합니다.

* **Min Confidence (최소 신뢰도 - 기본값 40):** 
  - 이 수치보다 낮은 결함은 아예 화면(썸네일)에 띄우지 않습니다. 
  - (예: 먼지가 너무 많이 잡히면 이 수치를 50으로 올리세요.)
* **Max Confidence (최대 신뢰도 - 기본값 90):** 
  - 이 수치보다 높은 결함은 명백한 결함(Auto-Reject)으로 간주하여 작업자 리뷰 화면에 띄우지 않습니다.
  - (예: 확실한 불량까지 작업자가 직접 두 눈으로 전부 다 보고 싶다면 100으로 설정하세요.)
* **IoU Threshold (겹침 허용치 - 기본값 0.45):** 
  - 동일한 결함에 여러 개의 박스가 겹쳐서 잡히는 것을 방지하는 수치입니다. 산업 표준인 0.45를 그대로 사용하는 것을 권장합니다.

> ⚠️ **주의:** 설정(Settings) 값을 변경하고 [Ok] 버튼을 누르면, **진행 중이던 리뷰 내역이 모두 초기화**되고 해당 폴더의 AI 연산을 처음부터 다시 수행합니다.

---

## 7. 판정 결과 저장하기

모든 결함에 대한 판정을 마쳤다면 결과를 파일로 저장해야 합니다.

1. 상단 메뉴에서 **[File] > [Save Review Results]** 를 클릭합니다.
2. 팝업 창이 뜨며 저장이 완료되었음을 알립니다.
3. 현재 열려있는 이미지 폴더 안에 `review_results.json` 파일이 생성됩니다.
4. 이 파일에는 작업자가 어떤 결함을 Pass/Fail 처리했는지에 대한 최종 데이터가 기록되며, 추후 AI 재학습이나 보고서 작성에 활용됩니다.
